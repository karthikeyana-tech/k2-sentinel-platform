# K² Sentinel Engineering Standards

## Philosophy

We build software that is:

- Simple
- Reliable
- Secure
- Observable
- Maintainable
- Testable

---

## Git Standards

- Small commits
- One feature per commit
- Conventional Commit Messages

Example:

feat(auth): add JWT authentication

---

## Code Standards

- Type hints required
- Docstrings required
- No duplicated logic
- Follow Clean Architecture
- Keep functions focused on a single responsibility

---

## API Standards

- REST First
- OpenAPI Documentation
- Versioned APIs
- Consistent Response Format

---

## Security

- Never hardcode secrets
- Use environment variables
- Validate all inputs

---

## Logging

- No print() statements
- Structured logging only

---

## Testing

Every feature must be testable.

---

## Documentation

Every major feature must include documentation.

---

## Engineering Motto

Design → Build → Test → Document → Deploy
